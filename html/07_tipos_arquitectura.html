Tipos de
Arquitectura
Front & Back

Los patrones arquitectónicos que definen cómo se estructura y organiza el software moderno.

Frontend
📄
SPA — Single Page App
React · Vue · Angular

Una sola página HTML que actualiza el DOM dinámicamente con JS. No recarga la página completa al navegar.

Browser ──▶ index.html (vacío) ──▶ bundle.js (toda la app) ◀── Renderiza en cliente

Úsalo en: Apps con mucha interacción, dashboards, herramientas tipo Gmail.

🖥️
SSR — Server Side Rendering
Next.js · Nuxt · Remix

El servidor genera el HTML completo en cada petición. Mejor SEO y tiempo de primera carga visible (FCP).

Request ──▶ Servidor Servidor genera HTML completo ◀── HTML listo (hidratado)

Úsalo en: E-commerce, blogs, apps con SEO crítico o usuarios lentos.

🧊
SSG — Static Site Generation
Astro · Next.js · Gatsby

Las páginas se generan en tiempo de build, no en runtime. Velocidad máxima, sin servidor. Sirve desde CDN.

Build time: datos ──▶ generador generador ──▶ HTML estáticos Runtime: CDN sirve archivos

Úsalo en: Portafolios, documentación, landing pages, blogs.

🏝️
Islands Architecture
Astro · Fresh · Qwik

Página mayormente estática con "islas" interactivas. Hidrata solo los componentes que lo necesitan.

🟦 HTML estático (0 JS) ⚡ Isla interactiva (JS mínimo) 🟦 HTML estático (0 JS) ⚡ Isla carrito (JS reactivo)

Úsalo en: Sitios de contenido con componentes interactivos puntuales.

🧩
Micro Frontends
Module Federation · single-spa

Divide el frontend en aplicaciones independientes. Cada equipo despliega su UI de forma autónoma.

Shell App ├── Team A: Checkout (React) ├── Team B: Catálogo (Vue) └── Team C: Perfil (Angular)

Úsalo en: Grandes organizaciones, portales corporativos con múltiples equipos.

📱
PWA — Progressive Web App
Service Worker · Web APIs

Web app con capacidades nativas: offline, notificaciones push, instalable en el dispositivo sin app store.

Service Worker intercepta ├── Cache offline-first ├── Push notificaciones └── Instalable en OS

Úsalo en: Apps con usuarios móviles, áreas con mala conectividad.

Backend
🧱
Monolítica
Django · Rails · Spring MVC

Toda la aplicación en un único proceso desplegable. Simple al inicio, puede volverse difícil de escalar.

[ UI + Negocio + Datos ] en un solo proceso DB ◀──▶ App ──▶ HTTP Deploy único ✓ Simple

Úsalo en: MVPs, equipos pequeños, apps de dominio simple.

⚡
Microservicios
Spring Boot · NestJS · FastAPI

Servicios pequeños e independientes, cada uno con su propia DB y proceso de despliegue autónomo.

Auth─┐ Pedidos─┤ ──▶ API Gateway Pagos──┘ Cada uno: DB propia

Úsalo en: Sistemas grandes, alta demanda, múltiples equipos.

☁️
Serverless / FaaS
AWS Lambda · Vercel · Cloudflare

Funciones que ejecutan bajo demanda. Sin gestión de servidores. Pagas solo lo que usas. Escala automáticamente.

Event ──▶ Lambda fn() fn() ejecuta ──▶ response Sin servidor activo Escala: 0 → ∞

Úsalo en: Procesos batch, webhooks, APIs de bajo tráfico.

🔄
Event-Driven
Kafka · RabbitMQ · EventBridge

Los servicios se comunican a través de eventos asíncronos. Máximo desacoplamiento entre productor y consumidor.

Productor ──▶ Event Bus Bus ──▶ Consumidor A Bus ──▶ Consumidor B Desacoplados: async

Úsalo en: Flujos de alta carga, integraciones complejas, auditoría.

📋
Hexagonal / Ports & Adapters
Clean Arch · DDD

El dominio de negocio al centro, sin dependencias externas. Adapters intercambiables: DB, UI, APIs.

[ HTTP ] [ DB ] ↕ (Adapters) ↕ [ Puerto ] [ Dominio ] [ Puerto ] ↕ (Adapters) ↕ [ CLI ] [ Queue ]

Úsalo en: Lógica de negocio compleja, alta testabilidad requerida.

📊
CQRS + Event Sourcing
Axon · EventStore · MediatR

Separa lecturas de escrituras. El estado se reconstruye reproduciendo eventos inmutables almacenados en orden.

Command ──▶ Write DB Evento generado ──▶ Store Store ──▶ Read DB (optimizada) Query ──▶ Read DB

Úsalo en: Auditoría, sistemas financieros, alta carga de lectura.
